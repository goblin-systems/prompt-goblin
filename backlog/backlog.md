## Operating Model

- Strategy defines direction.
- Release sequencing defines when work lands.
- Epic files define execution scope, dependencies, and acceptance.
- Stories and implementation tickets should roll out from the relevant epic file, not from this document.

## Current Execution Priorities

1. Reliability of capture and typing
2. Trust layer and recording controls
3. Per-app behavior and post-processing
4. Personal memory and local history
5. Adjacent workflow expansion

## Release Path

- `R0.2`: Daily-driver reliability
- `R0.3`: Personalized desktop dictation
- `R0.4`: Context-aware editing workflows

## Deferred Until Core Roadmap Proves Out

- full meeting mode with speaker separation, chaptering, summaries, and action items
- trust layer ghost mode beyond v1 preview and confirm flow
- wake phrase trigger
- screen-aware screenshot context
- fully local and offline mode as the primary near-term bet
- team packs
- broad cross-platform parity initiative
